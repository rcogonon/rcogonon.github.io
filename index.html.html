Profile

End-to-end analytics — from raw data to the stakeholder's decision

I own the full lifecycle of enterprise reporting on Microsoft Fabric and Power BI — from working with data engineers to shape curated Lakehouse tables, through Power Query ETL, semantic modeling, DAX, report design, performance tuning, and finally deployment through Power BI Apps with governed access. Every layer is treated as part of one solution, not an isolated task.

On the modeling side, I design shared semantic models on star-schema principles with reusable DAX measures — a single governed model that powers many reports, keeping business logic consistent. Rather than one dataset per report, I build once and deploy widely, with dynamic KPIs, hierarchies, drill-down paths, and self-service analytics baked in. Incremental refresh is configured at both the semantic model and dataset level so large fact tables stay fast without full reloads.

I work across all Power BI connectivity modes — Import, DirectQuery, Direct Lake, and composite — choosing by performance and business need. On governance, I implement Row-Level Security, manage workspaces in Power BI Service, configure scheduled refreshes through Fabric pipelines, and deploy via Power BI Apps with audience-based access. Reports come with built-in validation checks so inconsistencies get caught before they reach a stakeholder.

Bridging engineering discipline with modern analytics — so every dashboard ties back to a real decision on the ground.
